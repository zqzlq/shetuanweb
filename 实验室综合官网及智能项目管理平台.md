实验室综合官网及智能项目管理平台

# 一、项目简介

## 项目背景

为全面展示实验室整体形象、对外宣传科研实力，同时实现内部人员、项目、成果一体化数字化管理，搭建集__对外展示\+对内管控__于一体的实验室综合官网平台。平台融合对外宣传门户、成员管理、成果展示、招新入驻、智能项目管理、证书智能识别录入等功能，配套Web可视化界面与大模型专用CLI双接口，兼顾师生日常操作、系统自动化调用需求，解决实验室信息分散、成果录入繁琐、项目管理低效、对外宣传形式单一等问题，打造规范化、智能化、一体化的实验室线上门户与管理系统。

## 项目目标

1. 搭建完整实验室官网门户，完善对外展示板块，全面呈现实验室文化、团队、研究方向、成果荣誉、合作交流、招新信息等内容。
2. 实现内部人员权限管理、个人中心、证书智能上传识别功能，自动提取奖状信息，简化成果录入流程。
3. 集成智能项目管理智能体（双接口架构），支持自然语言需求梳理、任务拆分、周期规划、进度提醒、周报智能分析，完成实验室项目全流程管控。
4. 统一Web交互界面与大模型CLI命令行双接口，保障数据互通、功能同步，适配人工操作与自动化调用两类场景。
5. 实现网站稳定部署、数据安全存储，页面美观易用、内容可后台自主维护，满足长期运营使用需求。

## 核心功能概述

平台分为__对外展示门户、内部管理系统、个人中心、证书智能识别模块、智能项目管理智能体、双接口服务__六大板块。对外展示实验室风貌、团队、科研、奖项、招新等内容；内部实现人员、信息、项目统一管理；个人中心支持证书图片上传，依托图像识别自动填充获奖信息；项目管理智能体担任智能秘书，完成需求整理、任务排期、进度跟踪、周报分析与报告推送；Web界面供师生可视化操作，CLI接口面向大模型提供自动化调用能力。

## 应用场景

实验室对外形象宣传、访客信息查阅、师生日常办公、成果归档管理、科研/实训项目全周期管理、新生招新报名入驻。

# 二、项目任务书

## 项目基本信息

- 项目名称：实验室综合官网及智能项目管理平台
- 开发主体：学生开发团队
- 使用对象：实验室负责人、在职教师、在校学生、校外访客、意向入驻人员
- 架构模式：B/S架构，Web可视化交互界面 \+ 大模型专用CLI命令行双接口
- 核心定位：对外宣传窗口 \+ 对内综合管理平台 \+ 智能项目管控系统

## 总体开发任务

完成平台需求分析、架构设计、数据库搭建、前后端开发、智能识别功能、项目管理智能体、双接口开发、联调测试与部署上线。实现官网展示、人员管理、招新报名、证书智能录入、项目智能管理、消息提醒等全部功能，保证页面正常访问、功能稳定、接口可用、数据安全。

## 分模块详细任务清单

### 任务1：整体架构、数据库与接口规范设计

1. 规划系统整体架构，划分门户前台、管理后台、个人中心、智能识别引擎、项目管理智能体、CLI接口六大模块。
2. 设计全套数据表：访客信息表、实验室介绍表、成员信息表、研究方向表、成果奖项表、证书信息表、招新报名表、项目信息表、任务表、周报表、提醒记录表、用户权限表等。
3. 制定Web前后端接口、CLI命令行接口协议、数据格式标准，确保双接口数据实时同步。

### 任务2：实验室官网前台（对外展示板块开发）

1. __首页__：实验室简介、文化理念、风貌展示、核心公告、快速导航。
2. __实验室介绍__：发展历程、实验室精神、场地设施、组织架构。
3. __团队成员__：教师介绍、在读学生、往届校友，支持分类展示、信息查看。
4. __研究方向__：各科研方向说明、技术领域、研究内容、应用场景。
5. __成果展示__：科研项目、学术论文、专利、学科竞赛奖项、代表性作品，分类展示。
6. __合作交流__：校企合作、校际交流、学术活动、参会参展记录。
7. __招新/入驻通道__：入驻要求、报名入口、表单提交、状态查询。
8. __联系我们__：地址、联系方式、留言反馈功能。

### 任务3：后台管理系统开发

1. 权限管理：区分管理员、教师、普通学生权限，控制后台操作范围。
2. 内容管理：前台所有展示内容__后台增删改查__，无需改代码即可更新官网信息。
3. 成员管理：人员信息录入、编辑、分类、状态管理。
4. 成果管理：统一管理各类成果、奖项数据，支持批量操作。
5. 报名管理：查看、审核招新报名信息，反馈审核结果。
6. 系统设置：网站基础配置、轮播图、公告发布、数据备份。

### 任务4：个人中心 \+ 证书智能识别录入模块（重点功能）

1. 个人中心基础功能：个人信息查看、资料编辑、密码修改、我的成果、我的项目。
2. 证书上传功能：支持图片上传、预览、删除，限制文件格式与大小。
3. 证书智能识别引擎：上传奖状图片后，__自动识别获奖名称、颁奖单位、获奖时间、奖项等级__等字段，自动回填至对应表单。
4. 人工修正：识别内容支持手动编辑确认，提交后自动归档至个人成果库与实验室总成果库。

### 任务5：智能项目管理智能体（延续原有需求）

1. 自然语言需求解析：识别教师模糊需求，梳理标准化内容、指标、约束条件。
2. 智能任务规划：自动划分项目周期、里程碑、每周任务，支持手动调整。
3. 定时提醒：向教师推送进度审阅提醒，向学生推送任务、周报提交提醒。
4. 周报智能分析：解析学生周报，比对任务要求，判定完成度与进度，生成简易审核报告推送教师。

### 任务6：双接口开发（Web界面 \+ CLI命令行接口）

1. Web交互界面：整合官网、后台、个人中心、项目管理全场景可视化操作页面。
2. CLI专用接口：面向大模型开发命令行接口，支持信息查询、项目操作、数据获取、成果导出等指令调用。
3. 数据同步：双接口操作结果双向实时同步，数据一致无冲突。

### 任务7：系统联调、测试、优化与部署

1. 全模块联调，修复功能异常、页面兼容、数据错误等问题。
2. 分场景测试：官网访问测试、权限测试、证书识别测试、项目管理测试、双接口调用测试、兼容性测试。
3. 页面UI优化、加载速度优化、安全防护配置。
4. 完成服务器部署，配置域名、运行环境，上线运行。

## 人员分工

1. 架构&数据库组：整体架构设计、数据表设计、接口文档编写、数据规范制定。
2. 前端开发组：官网前台页面、管理后台页面、个人中心页面、交互效果实现。
3. 后端业务组：业务逻辑开发、权限控制、数据读写、消息推送、招新/成果管理。
4. 智能功能组：证书图像识别开发、项目管理智能体、周报分析、CLI接口开发。
5. 测试&运维组：功能测试、BUG修复、性能优化、系统部署、环境配置、使用文档编写。

2. 后台管理功能正常，可自主编辑、更新网站全部内容，人员、成果、报名审核流程顺畅。
