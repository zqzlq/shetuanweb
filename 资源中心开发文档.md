# 资源中心开发文档

> 社团内部综合资源平台，面向 50+ 多届成员，支持文件夹管理、多格式上传下载、在线预览、标签、搜索分页。

---

## 一、开发任务清单

### 1. 后端 - 数据模型

- [ ] 在 `backend/models.py` 中新增 `Resource` 模型
  - `id` Integer PK
  - `name` String(255) 显示名称
  - `is_folder` Boolean 区分文件夹/文件
  - `parent_id` FK→resources.id 自引用树结构
  - `tags` JSON 标签数组
  - `description` Text 描述
  - `file_url` String(500) 文件URL（仅文件）
  - `oss_key` String(500) OSS键名（便于删除）
  - `original_name` String(255) 原始文件名
  - `file_size` Integer 字节数
  - `mime_type` String(100) MIME类型
  - `file_ext` String(20) 扩展名（小写无点）
  - `uploader_id` FK→member_users.id 上传者
  - `status` String(20) pending/approved/rejected
  - `download_count` Integer 下载次数
  - `created_at` / `updated_at` DateTime
- [ ] 在 `backend/app.py` 中确保新表通过 `db.create_all()` 自动创建

### 2. 后端 - 用户端 API

在 `backend/routes/api.py` 中新增以下端点：

- [ ] `GET /api/resources/tree` - 获取文件夹树（根文件夹 + 子项计数）
- [ ] `GET /api/resources` - 列出文件夹内容（分页）
  - 参数：`parent_id`, `page=1`, `per_page=20`, `search`, `tag`
  - 响应：`{ items, total, page, per_page, pages }`
- [ ] `GET /api/resources/:id` - 获取单个资源详情
- [ ] `POST /api/resources/upload` - 上传文件
  - Form-data：file, parent_id, tags(JSON字符串), description
  - 流程：校验扩展名 → UUID文件名 → OSS上传 → 回退本地 → 创建记录
  - 常量：`ALLOWED_RESOURCE_EXTENSIONS`（文档/图片/压缩包/代码/音视频）
- [ ] `POST /api/resources/folder` - 创建文件夹
  - JSON：name, parent_id, description
- [ ] `GET /api/resources/:id/download` - 下载文件（download_count + 1，返回 file_url）
- [ ] `GET /api/resources/:id/preview` - 获取预览信息
  - image/* → 返回 URL
  - application/pdf → 返回 URL
  - text/* / .md / 代码文件 → 返回文本内容（限 <1MB）
  - 其他 → 返回文件元信息

### 3. 后端 - 管理端 API

- [ ] `GET /api/admin/resources` - 列出所有资源（分页，含 pending）
  - 参数：page, per_page, search, status, is_folder
- [ ] `PATCH /api/admin/resources/:id` - 修改元数据（name, tags, description, status）
- [ ] `DELETE /api/admin/resources/:id` - 删除资源（含 OSS 文件清理）
  - 非空文件夹禁止删除，返回错误
- [ ] `POST /api/admin/resources/:id/move` - 移动到目标文件夹
  - JSON：new_parent_id
- [ ] `PATCH /api/admin/resources/:id/status` - 审核状态变更
- [ ] `POST /api/admin/resources/batch` - 批量操作
  - JSON：ids, action(delete/approve/reject/move), new_parent_id

## 五、关键实现细节

### 认证复用
- 用户端：`@jwt_required()` + `identity.startswith('user_')` + `MemberUser.status == 'active'`
- 管理端：`@jwt_required()` + identity 为纯数字 + `AdminUser.query.get(int(identity))`

### 存储复用
- OSS：`oss_service.upload_file(key, bytes)` / `oss_service.delete_file(key)`
- 本地回退：写入 `UPLOAD_FOLDER/resources/`，返回 `/uploads/resources/{filename}`
- 存储 oss_key 字段，避免从 URL 解析

### 分页
- 使用 SQLAlchemy `paginate(page=page, per_page=per_page)`
- 响应格式：`{ items, total, page, per_page, pages }`

### 文件夹删除
- 非空文件夹禁止删除，返回 `{ error: 'not_empty', message: '文件夹内有 N 个子项，请先清空' }`

### 面包屑
- 后端在列表响应中返回 ancestor 链（沿 parent_id 向上遍历）
- 前端渲染为 `根目录 / 文件夹A / 文件夹B`

### 标签过滤
- 存储为 JSON 数组
- 使用 `Resource.tags.contains('"tagName"')` 做字符串包含匹配
